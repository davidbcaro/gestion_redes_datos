GA4 · Simulación en Packet Tracer

GA4-220501104-AA1-EV01 · 220501104

Implementación de un esquema de subredes y direccionamiento IPv4 e IPv6 con Packet Tracer 9.0. Caso Enter Ltda., cuatro VLAN, router on a stick y cálculo VLSM a partir de 192.168.1.0/24 y prefijo IPv6 2001:DB8:ACAD::/48. A continuación encontrará el marco teórico y la guía práctica; al final, los archivos originales (Word, PowerPoint y Excel) de la carpeta.

1. Teoría 2. Práctica 3. Glosario 4. Archivos de apoyo

Marco teórico

Objetivo y entregables

Debe construir en Packet Tracer la red de la empresa a partir del espacio 192.168.1.0/24, con subnetting VLSM y direcciones IPv6 a partir de 2001:DB8:ACAD::/48 (un /64 por VLAN). Los entregables habituales son:

  • Archivo de simulación .pkt con topología probada (por ejemplo Enter_Ltda.pkt).
  • Hoja de cálculo con VLSM y tablas de direccionamiento IPv4 e IPv6.
  • Evidencias de conectividad: capturas de ping entre VLAN y salida de show ip interface brief (y equivalentes IPv6).

Plataforma. La entrega se realiza en Zajuna según el enlace y la fecha que indique su instructor. Verifique nombres y formatos requeridos.

Escenario: Enter Ltda.

Se segmenta la LAN en cuatro áreas (cada una con su VLAN y subred). Un único Cisco 2901 enruta entre VLAN mediante router on a stick: subinterfaces 802.1Q sobre GigabitEthernet0/0 hacia un switch 2960.

IPv4: VLSM

VLSM (máscaras de subred de longitud variable) asigna un prefijo distinto a cada subred según el número real de hosts. Se parte de 192.168.1.0/24 y se asignan las subredes de mayor a menor tamaño.

Regla de oro VLSM: asigne siempre primero la subred más grande. Si empieza por la más pequeña, el siguiente bloque puede no alinearse y el diseño deja de ser válido.

IPv6: prefijos por VLAN

Se utiliza el prefijo global 2001:DB8:ACAD::/48 y se reserva un /64 por VLAN. Cada VLAN se refleja en el cuarto hexteto (:10, :20, :30, :40) para facilitar la lectura en laboratorio y cursos CCNA.

Reglas de notación IPv6: elimine ceros a la izquierda en cada hexteto; use una sola vez :: para el bloque más largo de grupos a cero; toda interfaz activa requiere dirección link-local (FE80::/10). Usar FE80::1 en todas las subinterfaces del router es una convención didáctica: el host siempre alcanza el mismo “gateway” link-local en el enlace local.

Packet Tracer (contexto)

Packet Tracer es el simulador de Cisco Networking Academy. La versión 9.0 mejora IoT, IPv6 y otras funciones. Se descarga desde netacad.com (recursos de la cuenta; curso Introduction to Packet Tracer si no tiene otra ficha). En el primer arranque inicie sesión: sin cuenta, el programa deja de funcionar a los 15 minutos. Los .pkt creados en 9.x no se abren en 7.x; si su aula pide otra versión, use File → Save As a un formato compatible (p. ej. 8.2) si aplica.

Parte práctica

1. Topología básica

  1. Coloque un Cisco 2901 (nombre R1) y un 2960 (nombre SW1).
  2. Al menos un PC por VLAN: p. ej. PC-ADM1, PC-FIN1, PC-TEC1, PC-VIS1.
  3. Cablee con cobre recto: R1 G0/0 ↔ SW1 Fa0/1 (troncal); asigne puertos de acceso según su diseño (p. ej. Fa0/2 a VLAN 10, etc.).

Consejo: use nombres descriptivos, active View → Port Labels si le ayuda a documentar, y guarde a menudo (Ctrl+S).

2. Switch SW1: VLAN, trunk y acceso

Cree las cuatro VLAN con nombre, más una VLAN 99 nativa (práctica habitual frente a riesgos de VLAN hopping en tráfico con etiquetado inadecuado). Trunk hacia R1 y puertos de acceso por puerto.

SW1> enable
SW1# configure terminal
SW1(config)# hostname SW1
SW1(config)# vlan 10
SW1(config-vlan)# name ADMINISTRATIVA
SW1(config-vlan)# vlan 20
SW1(config-vlan)# name FINANCIERA
SW1(config-vlan)# vlan 30
SW1(config-vlan)# name TECNICOS
SW1(config-vlan)# vlan 40
SW1(config-vlan)# name VISITANTES
SW1(config-vlan)# vlan 99
SW1(config-vlan)# name NATIVA
SW1(config)# interface FastEthernet0/1
SW1(config-if)# switchport mode trunk
SW1(config-if)# switchport trunk allowed vlan 10,20,30,40,99
SW1(config-if)# switchport trunk native vlan 99
SW1(config-if)# description Trunk hacia R1 G0/0
SW1(config-if)# interface range FastEthernet0/2 - 24
SW1(config-if-range)# switchport mode access
SW1(config-if-range)# interface FastEthernet0/2
SW1(config-if)# switchport access vlan 10
SW1(config-if)# interface FastEthernet0/3
SW1(config-if)# switchport access vlan 20
SW1(config-if)# interface FastEthernet0/4
SW1(config-if)# switchport access vlan 30
SW1(config-if)# interface FastEthernet0/5
SW1(config-if)# switchport access vlan 40
SW1(config-if)# end
SW1# copy running-config startup-config

3. Router R1: IPv4 (router on a stick)

Active la interfaz física, luego una subinterfaz por VLAN con encapsulation dot1Q <id> (antes de la ip address). Puede añadir G0/0.99 con encapsulation dot1Q 99 native para alinear la VLAN nativa del trunk.

Router> enable
Router# configure terminal
Router(config)# hostname R1
R1(config)# ipv6 unicast-routing
R1(config)# interface GigabitEthernet0/0
R1(config-if)# description Trunk hacia SW1
R1(config-if)# no shutdown
R1(config-if)# interface GigabitEthernet0/0.10
R1(config-subif)# description ADMINISTRATIVA
R1(config-subif)# encapsulation dot1Q 10
R1(config-subif)# ip address 192.168.1.193 255.255.255.224
R1(config-subif)# interface GigabitEthernet0/0.20
R1(config-subif)# description FINANCIERA
R1(config-subif)# encapsulation dot1Q 20
R1(config-subif)# ip address 192.168.1.129 255.255.255.192
R1(config-subif)# interface GigabitEthernet0/0.30
R1(config-subif)# description TECNICOS
R1(config-subif)# encapsulation dot1Q 30
R1(config-subif)# ip address 192.168.1.225 255.255.255.224
R1(config-subif)# interface GigabitEthernet0/0.40
R1(config-subif)# description VISITANTES
R1(config-subif)# encapsulation dot1Q 40
R1(config-subif)# ip address 192.168.1.1 255.255.255.128
R1(config-subif)# interface GigabitEthernet0/0.99
R1(config-subif)# description NATIVA
R1(config-subif)# encapsulation dot1Q 99 native
R1(config-subif)# end

Orden en IOS: encapsulation dot1Q antes de ip address. De lo contrario, corregir la VLAN en una subinterfaz ya creada puede obligar a borrarla y volver a crearla.

4. Router R1: IPv6

En cada subinterfaz añada la dirección global ipv6 address …/64 y, si aplica, ipv6 address FE80::1 link-local (ya habilitó ipv6 unicast-routing a nivel global).

R1(config)# interface GigabitEthernet0/0.10
R1(config-subif)# ipv6 address 2001:DB8:ACAD:10::1/64
R1(config-subif)# ipv6 address FE80::1 link-local
R1(config-subif)# interface GigabitEthernet0/0.20
R1(config-subif)# ipv6 address 2001:DB8:ACAD:20::1/64
R1(config-subif)# ipv6 address FE80::1 link-local
R1(config-subif)# interface GigabitEthernet0/0.30
R1(config-subif)# ipv6 address 2001:DB8:ACAD:30::1/64
R1(config-subif)# ipv6 address FE80::1 link-local
R1(config-subif)# interface GigabitEthernet0/0.40
R1(config-subif)# ipv6 address 2001:DB8:ACAD:40::1/64
R1(config-subif)# ipv6 address FE80::1 link-local
R1(config-subif)# end
R1# copy running-config startup-config

5. PCs: direcciones de ejemplo

En cada PC: Desktop → IP Configuration (o pestaña Config): IPv4/IPv6 estáticos según su VLAN, gateway IPv4 = IP del router en esa VLAN; gateway IPv6 = FE80::1 (o la global del router, según lo que pida su guía).

6. Interfaces pasivas (OSPF o EIGRP, si aplica en su evidencia)

Evite enviar hellos de enrutamiento hacia las LAN de usuarios donde no hay vecinos. Con OSPF (proceso 1 de ejemplo):

R1(config)# router ospf 1
R1(config-router)# passive-interface GigabitEthernet0/0.10
R1(config-router)# passive-interface GigabitEthernet0/0.20
R1(config-router)# passive-interface GigabitEthernet0/0.30
R1(config-router)# passive-interface GigabitEthernet0/0.40
R1(config-router)# end

Alternativa con EIGRP AS 100: passive-interface default y luego deje activas solo las interfaces hacia vecinos reales, si su diseño lo incluye.

7. Comprobación

En R1: show ip interface brief, show ipv6 interface brief, show ipv6 route (o las variantes de su guía). Desde un PC, Command Prompt: pruebe ping IPv4 a su gateway, a otro host en otra VLAN y ping 2001:DB8:ACAD:20::10 (IPv6 inter-VLAN). tracert a otra subred confirma el salto por el router. Éxito: ping sin pérdida y ruta con el gateway esperado.

# Desde el PC: Command Prompt
ping 192.168.1.193
ping 192.168.1.130
ping 192.168.1.226
ping 192.168.1.10
ping 2001:DB8:ACAD:20::10
tracert 192.168.1.10

Orden de diagnóstico: 1) show vlan brief en SW1, 2) show interfaces trunk, 3) show ip interface brief en R1, 4) ipconfig en el PC. Si el simulador queda en estado raro, a veces conviene rehacer R1 y SW1 desde cero.

Glosario breve

TérminoSignificado
VLANLAN virtual: conjunto lógico de puertos de switch en la misma red de capa 2.
TrunkEnlace que transporta varias VLAN con etiquetado 802.1Q.
802.1QNorma IEEE que añade una etiqueta de 4 bytes en Ethernet para la VLAN.
VLSMMáscaras de longitud variable; optimiza el uso de espacio de direcciones.
Router on a stickUn router enruta entre VLAN por subinterfaces sobre un único enlace troncal.
SubinterfazInterfaz lógica (p. ej. G0/0.10) con su VLAN y su IP (IPv4/IPv6).
GatewayIP del router en la subred; los hosts la usan para alcanzar otras redes.
Link-local (IPv6)FE80::/10, válida solo en el enlace. Suele acompañar a toda interfaz activa.
Passive interfaceEl protocolo de enrutamiento no envía hellos por esa interfaz (mayor control en borde LAN).
.pktFormato de archivo de simulación de Packet Tracer.
